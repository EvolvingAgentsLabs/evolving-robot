EvolvingAgentsLabs · working example

A robot that rewrites its own skills — and versions every mutation.

A 2D patrol agent runs on Gemma 4 alone. It patrols a simulated facility, dreams about its failures, rewrites a skill, proves the edit is safe, and keeps or rolls it back by score. Four open tools, one closed loop, zero hardware.

02

Four tools, four jobs

the brain — plans · pilots · dreams Gemma 4 · Google AI Studio (REST)

One model does everything over a plain HTTPS call — no GPU, no downloads. Native function-calling drives the robot's primitives; the same model rewrites the skills. Confirmed working on gemma-4-26b-a4b-it.

the body + arena odyssey + sim2d

Mission orchestration and scoring. A custom Sim2DRunner composes a Gemma planner + pilot through odyssey's multi-agent runtime, driving a 2D simulator over WebSocket.

the safety gate skill-map

A semantic linter over the skill graph. Before any self-edit lands, sm check rejects broken cross-references, name collisions, and schema violations.

the genetic memory agentvcs

Multidimensional version control. Commits pin code + goal + the mission trace together; a regression triggers rollback, a verified win gets crystallized.

05

Built in the same workspace: agentvcs, improved

Two contributions surfaced by dogfooding

Using agentvcs to version a live evolving agent exposed two gaps. Both were implemented and tested in the sibling repo.

  • rollbackRepository.rollback(reason=…) — records why a rollback happened (an eval regression) in the durable ledger, not just the restored goal. Backward-compatible.
  • providerThe odyssey trace provider — reads odyssey's native missions.db with only the standard library and normalizes a mission (objective + per-task result_summary + grade) into agentvcs's message list.
